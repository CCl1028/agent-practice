## 1. 项目概述

### 1.1 产品定位

一个基于 **多 Agent 架构** 的基金投资助手 MVP，核心价值是：

> "你的基金，我盯着。有事我叫你，没事别操心。"

### 1.2 核心功能

| 功能模块 | 描述 | 状态 |
|---------|------|------|
| 持仓决策助手 | 每日分析持仓，给出加仓/减仓/观望/持有建议 | ✅ 已实现 |
| 多维技术指标 | 均线排列、乖离率、波动率、置信度评分 | ✅ v2 新增 |
| 新闻情报搜索 | 4 引擎实时搜索基金专属新闻 + 市场新闻 | ✅ v2 新增 |
| 交易纪律系统 | 追高检测、空头禁加仓、下跌禁补仓硬规则 | ✅ v2 新增 |
| 多源数据获取 | AKShare + Efinance 自动切换 + 熔断机制 | ✅ v2 新增 |
| 多渠道推送 | 微信/企业微信/Bark 推送每日简报 | ✅ 已实现 |
| 基金买前诊断 | 评估一只基金是否值得买（后期迭代） | 🚧 待开发 |
| 涨跌原因分析 | 解释今天为什么涨/跌 | 🚧 待开发 |

### 1.3 技术栈

```
┌─────────────────────────────────────────────────────────────┐
│                        技术选型                              │
├─────────────┬───────────────────────────────────────────────┤
│ Agent 框架  │ LangGraph (基于 LangChain)                     │
│ 后端框架    │ FastAPI                                        │
│ LLM 模型    │ DeepSeek-chat (可切换 OpenAI GPT)              │
│ 行情数据    │ AKShare (主) + Efinance (备) + Mock 兜底       │
│ 新闻搜索    │ Tavily / 博查 / Brave / SerpAPI (4引擎切换)    │
│ 持仓存储    │ JSON 文件 (MVP 阶段)                           │
│ 推送渠道    │ Server酱 / 企业微信 Webhook / Bark             │
│ 定时任务    │ APScheduler (应用内调度)                        │
│ 部署方式    │ Docker + docker-compose                        │
└─────────────┴───────────────────────────────────────────────┘
```

## 5. 状态管理 (LangGraph State)

### 5.1 核心 State 定义

```python
# src/state.py

class FundHolding(TypedDict):
    """单只基金持仓"""
    fund_code: str          # 基金代码 (6位数字)
    fund_name: str          # 基金名称
    cost: float             # 持仓金额 (元)
    cost_nav: float         # 成本净值
    current_nav: float      # 当前净值
    profit_ratio: float     # 盈亏比例 (%)
    hold_days: int          # 持有天数
    trend_5d: list[float]   # 近5日涨跌幅 (%)

class MarketData(TypedDict):
    """市场数据"""
    sectors: list[SectorData]    # 板块涨跌
    market_sentiment: str        # 市场情绪
    hot_news: list[str]          # 热点新闻

class Briefing(TypedDict):
    """最终简报输出"""
    summary: str                 # 一句话结论 (≤15字)
    details: list[FundAdvice]    # 每只基金的建议
    market_note: str             # 市场简评

class AgentState(TypedDict, total=False):
    """LangGraph 全局 State"""
    trigger: Literal["daily_briefing", "user_query", "new_portfolio"]
    user_query: str
    holdings: list[FundHolding]   # 前端传入 (可选)
    portfolio: list[FundHolding]  # Portfolio Agent 填充
    market: MarketData            # Market Agent 填充
    briefing: Briefing            # Briefing Agent 填充
    error: str
```

### 6.2 决策规则引擎详解 (v2)

```
决策维度: 盈亏 × 趋势 × 乖离率 × 均线 × 波动率 × 持有时间

返回值: (action, reason, score, risk_note)
  - action: 加仓/减仓/观望/持有
  - score: 0-100 置信度评分
  - risk_note: 风险提示（可为空）
```

#### 交易纪律硬规则（最高优先级）

| 规则 | 条件 | 结果 | 评分 |
|------|------|------|------|
| 🚫 追高检测 | 乖离率 > 3% 且上涨趋势 | 观望 + 追高风险提示 | 35 |
| 🚫 空头禁加仓 | 均线空头排列 且 浮亏 > 5% | 观望 + 趋势未反转 | 30 |
| ✅ 止盈不犹豫 | 浮盈 > 15% 且连涨 | 减仓（部分止盈） | 80 |

#### 常规决策矩阵

```
                │  上涨趋势     │  下跌趋势      │  震荡
  ──────────────┼───────────────┼────────────────┼──────────────
  浮盈 > 10%    │  减仓(65-75)  │  持有(55)      │  持有(55)
  浮亏 > 10%    │  加仓*(50-55) │  观望(40)⚠️    │  加仓*(50-55)
  盈亏 ≤ 10%    │  持有/观望    │  观望(45)      │  观望(45)

  * 加仓条件: 均线非空头排列 + 已企稳
  ⚠️ 风险: 仍在下跌，切勿补仓
```

#### 辅助信号

| 信号 | 条件 | 影响 |
|------|------|------|
| 盘中估值大跌 | est_change < -2% | 建议观望等日内企稳 |
| 盘中估值上涨 | est_change > 2% 且浮盈 > 5% | 短期偏强，继续持有 |
| 高波动警告 | 5日波动率 > 4% | 添加风险提示 |
| 深度套牢 | 浮亏 > 15% 且持有 > 180天 | 小额补仓摊低成本 |

| 术语 | 解释 |
|-----|------|
| NAV | Net Asset Value，基金净值 |
| 估值 | 盘中根据持仓股票走势预估的净值 |
| 趋势 | 近5个交易日的涨跌幅序列 |
| MA5/MA10/MA20 | 5/10/20 日移动平均线净值 ★ |
| 多头排列 | MA5 > MA10 > MA20，趋势向上 ★ |
| 空头排列 | MA5 < MA10 < MA20，趋势向下 ★ |
| 乖离率 | 现价偏离 MA5 的百分比，衡量追高风险 ★ |
| 波动率 | 近5日涨跌幅的最大值减最小值 ★ |
| 熔断器 | 数据源连续失败后自动冷却机制 ★ |
| 规则引擎 | 基于多维指标的决策逻辑（v2: 6维度） |
| 降级 | LLM/数据源不可用时使用备选方案 |
| 置信度评分 | 0-100 分量化规则引擎的判断信心 ★ |
