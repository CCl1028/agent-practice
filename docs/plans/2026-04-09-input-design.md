# 输入方案设计：截图录入持仓

> 日期：2026-04-09
> 状态：已确认

## 一句话总结

用户截一张持仓详情页截图 → 系统自动提取并录入 → 剩下的全靠 API 补全。零追问、零手动输入。

## 设计原则

- **极简输入**：一张截图搞定，不追问、不让用户填表
- **信息最大化**：截图提供持仓锚点（30%），系统 API 自动补全分析数据（70%）
- **兼容多 App**：支付宝、天天基金、招行、理财通、工行等主流 App 的截图都能识别

## 输入流程

### 场景 A：单只基金录入（推荐）

```
用户：[上传一张持仓详情页截图]

系统：✅ 识别到 易方达蓝筹精选(005827)
      持有 ¥32,450 | 成本净值 1.5716 | 收益率 -8.23%
      [确认录入] [修改] [继续添加]
```

### 场景 B：批量录入

用户连续截多张持仓详情页 → 逐张识别确认 → 全部入库。

或截一张**持仓列表页**（多只基金一览）→ 批量识别 → 缺成本净值的用公式反算。

## 截图提取字段

### 持仓详情页（信息最全，推荐）

| 字段 | 提取可靠度 | 说明 |
|------|-----------|------|
| 基金代码 | ★★★★★ | 6 位数字，所有 App 都显示 |
| 基金名称 | ★★★★★ | 大字显示 |
| 持仓金额/市值 | ★★★★★ | 大字显示 |
| 持有收益率 % | ★★★★★ | 红绿色标注 |
| 持有收益（元） | ★★★★☆ | 绝大多数 App 显示 |
| 持有份额 | ★★★★☆ | 多数 App 显示 |
| 成本净值 | ★★★☆☆ | 部分 App 不直接显示，需反算 |
| 最新净值 | ★★★★☆ | 页面上有，但 API 获取更准 |
| 日涨跌幅 | ★★★★☆ | 通常紧跟净值 |

### 持仓列表页（批量快捷，信息少一些）

| 字段 | 提取可靠度 | 说明 |
|------|-----------|------|
| 基金代码/名称 | ★★★★★ | 列表显示 |
| 持仓金额 | ★★★★★ | 列表显示 |
| 收益率 % | ★★★★☆ | 列表显示 |
| 成本净值 | ❌ | 列表页通常不显示，需反算 |

## 成本净值反算策略

当截图中未直接显示成本净值时，使用以下公式自动补全：

```
方法 1（优先）：
  成本净值 = 当前净值 × (1 - 收益率)

  例：当前净值 1.4521，收益率 -8.23%
  成本净值 = 1.4521 × (1 + 0.0823) ≈ 1.5716

方法 2（更精确，需要份额）：
  成本净值 = (持仓金额 - 收益金额) ÷ 持有份额
```

## 信息来源分层

截图只解决"用户持有什么"，分析数据全靠系统自动获取：

| 来源 | 提供的信息 | 占决策信息量 |
|------|-----------|------------|
| **截图提取** | 基金代码、名称、持仓金额、成本净值、收益率、份额 | ~30% |
| **API 自动补全** | 最新净值、近 5/20 日涨跌、MA 均线、乖离率、波动率、盘中估值、板块行情、市场情绪、基金/行业新闻 | ~70% |

## 决策所需信息全景

### ① 用户持仓状态（截图提供）
- 基金代码/名称
- 持仓金额
- 当前收益率
- 成本净值（截图提取或反算）
- 持有份额

### ② 基金走势（API 自动获取）
- 最新净值 + 日涨跌
- 近 5 日涨跌幅序列
- 近 20 日净值数据
- MA5 / MA10 / MA20 均线及排列状态
- 乖离率、波动率
- 盘中实时估值

### ③ 市场环境（API 自动获取）
- 大盘指数涨跌（上证/深证/创业板）
- 基金所属板块今日表现
- 整体市场情绪

### ④ 新闻情报（搜索 API）
- 基金/重仓股近期新闻
- 所属行业政策/事件

## 现有代码对接

基于 `agent-practice/src/tools/ocr_tools.py` 的 `process_screenshot` 函数：

1. **优化 Prompt**：让多模态 LLM 区分"持仓详情页"和"持仓列表页"两种截图格式
2. **增加字段提取**：详情页提取时增加 `cost_nav`（成本净值）、`shares`（份额）
3. **反算逻辑**：当成本净值缺失时，自动用收益率反算
4. **列表页批量**：列表页截图返回多只基金的数组
5. **确认流程**：识别后展示给用户确认，再入库

## 不做的事情（YAGNI）

- ❌ 不追问用户任何问题
- ❌ 不采集用户风险偏好/投资目标
- ❌ 不要求用户手动输入持有天数
- ❌ 不做自动导入（券商 API 接入）
