# 基金投资助手 MVP — 优化设计方案

> 日期：2026-03-31
> 状态：设计确认

---

## 一、产品定位

### 一句话定义

> "你的基金，我盯着。有事我叫你，没事别操心。"

### 核心差异化

| 维度 | 传统基金App | 本产品 |
|------|-----------|--------|
| 交互模式 | 用户主动查 | 系统主动推 |
| 信息密度 | 一堆数字图表 | 一句话结论 |
| 决策支持 | 给你数据，自己判断 | 直接告诉你"动不动" |
| 录入成本 | 手动填表 | 截图/说句话就行 |
| 使用频率 | 用户焦虑时打开 | 该看的时候才出现 |

### "懒人三不"原则

- **不需要主动打开** — 有事推送通知你
- **不需要看懂数据** — 结论前置，一句话说清
- **不需要自己判断** — Agent 帮你做决策

### 产品核心循环

1. **一次录入** — 截图或自然语言，30秒完成持仓录入
2. **每日简报** — Agent 主动分析，推送"今日一句话"
3. **按需深聊** — 想了解更多时，对话追问（后期迭代）

## 二、MVP 功能设计

### 模块1：智能持仓录入

**功能目标：** 用最低成本把用户持仓数据搞进来。

**支持两种方式：**

#### 方式A：截图识别（主）
- 用户截一张基金 App 的持仓页截图
- GPT-4o Vision 直接解析，提取：基金代码、基金名称、持有金额、成本价、当前收益率
- 解析后让用户确认，一键保存

#### 方式B：自然语言描述（辅）
- 用户说："我买了2万的易方达蓝筹，大概去年6月买的"
- LLM 提取关键信息 + 自动补全数据（根据历史净值估算成本价）

**MVP 要求：**
- 截图识别准确率 > 90%
- 录入全流程 < 30秒

---

### 模块2：每日操作简报（核心）

**功能目标：** 每天告诉用户"今天要不要动"。

**输出格式 — 三层递进：**

```
【第一层：推送通知 — 1秒看完】
📊 今日持仓：无需操作 ✅

【第二层：打开简报 — 10秒看完】
┌─────────────────────────────────┐
│  易方达蓝筹  -5.6%  👉 观望     │
│  招商中证白酒 +12.3% 👉 可减仓  │
│                                 │
│  市场：科技回调，消费走强        │
│  一句话：今天别动，等信号        │
└─────────────────────────────────┘

【第三层：点击展开 — 想看再看】
详细理由、行业分析、趋势数据...
```

**核心原则：结论前置，理由按需。**

**决策规则（MVP 版本）：**
- 浮盈 > 10% 且短期连涨 → 偏减仓
- 浮亏 > 10% 且企稳 → 可考虑加仓
- 无明显趋势 → 观望
- Agent 负责综合多因素，生成自然语言解释

---

### 模块3：涨跌原因分析

**功能目标：** 在简报中解释"今天为什么涨/跌"。

**实现方式：**
- 获取基金重仓行业的板块涨跌
- 获取当日关键市场新闻
- LLM 综合生成原因分析

**输出示例：**
```
易方达蓝筹今日 -1.2%
原因：所属科技板块整体回调，重仓股表现偏弱
结论：板块波动，暂未看到基本面变化
```

---

### 模块4：基金买前诊断（可选）

**功能目标：** 回答"这只基金值不值得买？"

**输入：** 基金名称/代码

**输出：**
```
综合评价：中等偏上

优点：近一年收益较好，行业集中在成长赛道
风险：波动较大，行业集中度较高
建议：可以关注，建议分批建仓
```

**MVP 重点：** 结构化表达 + 解释能力，不追求专业量化。

---

### 后期迭代：Chat 入口

- 保留对话交互入口，MVP 之后迭代
- 支持用户主动提问："我的XX基金怎么样"、"这只值得买吗"
- Supervisor Agent 识别意图后路由到对应 Agent

## 三、技术架构

### 多 Agent 架构设计

```
                    ┌──────────────┐
                    │  Supervisor  │  协调者：调度任务、合并结果
                    │   Agent      │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
     ┌────────────┐ ┌───────────┐ ┌────────────┐
     │  Portfolio  │ │  Market   │ │  Briefing  │
     │  Agent     │ │  Agent    │ │  Agent     │
     │ 持仓管家    │ │ 市场观察员 │ │ 简报撰写员  │
     └────────────┘ └───────────┘ └────────────┘
```

### Agent 职责

| Agent | 职责 | 绑定工具 |
|-------|------|---------|
| **Supervisor** | 接收触发（定时/用户提问），决定调用哪些 Agent，合并结果 | 路由逻辑 |
| **Portfolio Agent（持仓管家）** | 管理持仓数据、解析截图/自然语言、计算盈亏状态 | OCR（GPT-4o Vision）、数据库读写、净值计算 |
| **Market Agent（市场观察员）** | 获取实时行情、行业动态、板块涨跌、市场新闻 | AKShare API、新闻摘要 |
| **Briefing Agent（简报撰写员）** | 接收前两个 Agent 的数据，生成结论和建议 | LLM 推理、模板格式化 |

### 关键架构优势

1. **并行执行** — Portfolio Agent 和 Market Agent 同时工作，不串行等待
2. **条件路由** — Supervisor 根据场景决定流程（日常简报 vs 新持仓录入，走不同路径）
3. **工具隔离** — 每个 Agent 绑定自己的工具集，职责清晰
4. **易扩展** — 后续加 Agent（如"定投顾问"）只需注册到 Supervisor

### 数据流 — 三种触发场景

```
场景1：每日简报（定时触发）
  Supervisor
    ├─→ Portfolio Agent（读取已有持仓，计算最新盈亏）  ──┐
    ├─→ Market Agent（拉取今日行情、板块、新闻）       ──┤ 并行
    │                                                    ▼
    └─→ Briefing Agent（合并数据，生成简报）→ 推送用户

场景2：用户提问（Chat 触发，后期迭代）
  Supervisor（分析意图）
    ├─ "我的XX基金怎么样" → Portfolio + Market + Briefing
    ├─ "这只基金值得买吗" → Market + Briefing（无需持仓）
    └─ "今天为什么跌了"   → Market + Briefing（无需持仓）

场景3：新持仓录入
  Supervisor
    └─→ Portfolio Agent（解析截图/文本 → 提取 → 存储）
        └─→ 自动触发一次场景1，给出首次建议
```

## 四、技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent 框架 | LangGraph | 适合多 Agent DAG，支持并行和条件路由 |
| 后端 | FastAPI | 轻量、异步、适合 API 服务 |
| 模型 | GPT-4o-mini（文本）+ GPT-4o（截图识别） | MVP 性价比最高 |
| 数据源 | AKShare（免费）+ mock 兜底 | 国内基金数据免费库 |
| 存储 | SQLite | MVP 够用，后期可迁移 |
| 前端 | Streamlit | 最快出活 |
| 推送 | Server酱 / 企业微信 Webhook | 零成本实现主动推送到微信 |

## 五、开发排期（3周）

| 周 | 目标 | 交付物 |
|----|------|--------|
| **第1周** | 跑通核心链路 | 3个 Agent 串通，mock 持仓 → 输出简报文本，LangGraph DAG 可运行 |
| **第2周** | 打通输入层 | 截图 OCR 解析 + 自然语言持仓录入，SQLite 持久化，接入 AKShare 真实数据 |
| **第3周** | 产品化 | Streamlit 简报页面、推送到微信、输出格式打磨、Prompt 调优 |

**后续迭代：**
- Chat 对话入口
- 定投建议模块
- 多用户支持

## 六、验收标准

- ✅ 截图一张持仓图，30秒内完成识别并给出建议
- ✅ 每日简报推送到手机（微信/企业微信）
- ✅ 简报第一层结论 ≤ 15个字
- ✅ 3个 Agent 有明确的并行执行（不是伪并行）
- ✅ 用户输入后能得到明确建议
- ✅ 输出结构清晰（结论 + 理由 + 操作）
- ✅ 每次结果都"看起来合理"
- ✅ 有一定解释能力

---

## 七、核心总结

这个 MVP 的本质不是"基金查询工具"，而是：

> **一个帮你盯盘、帮你判断、只在该说话时说话的私人基金管家。**

用户不需要懂投资、不需要看数据、不需要做决策。Agent 替你完成"信息收集 → 分析判断 → 结论输出"的完整链路。
