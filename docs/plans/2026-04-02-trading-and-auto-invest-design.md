# 加减仓 & 定投计划功能设计

> 日期：2026-04-02
> 状态：已确认，待实施

---

## 一、功能概述

在现有持仓管理和每日简报的基础上，新增三个核心能力：

1. **加减仓交易记录** — 用户可对已有基金进行加仓/减仓操作，每笔交易留痕
2. **定投计划** — 用户为基金设置自动定投，系统按周期模拟执行并更新持仓
3. **累积持仓计算** — 根据所有交易记录（手动 + 定投）自动计算真实的持仓成本和收益

### 设计决策

| 决策项 | 结论 |
|--------|------|
| 数据存储 | 先 localStorage，后续再考虑同步 |
| 加减仓交互 | 卡片按钮 + 自然语言，两种都支持 |
| 定投执行方式 | 自动模拟执行（非真实交易，是记账） |
| 定投频率 | MVP 固定周期，智能定投留接口 |
| 持仓展示 | 卡片简洁 + 点击展开完整交易流水 |

## 二、数据模型

### 2.1 新增：Transaction（交易记录）

```
Transaction
├── id: string              # 唯一ID（时间戳+随机数）
├── fund_code: string       # 基金代码
├── type: "buy" | "sell"    # 买入/卖出
├── amount: number          # 交易金额（元）
├── nav: number             # 成交净值（当日净值）
├── shares: number          # 份额 = amount / nav
├── source: "manual" | "auto_invest"  # 来源：手动操作 / 定投自动
├── created_at: string      # 时间戳
└── note: string            # 备注（可选）
```

### 2.2 新增：AutoInvestPlan（定投计划）

```
AutoInvestPlan
├── id: string              # 唯一ID
├── fund_code: string       # 基金代码
├── fund_name: string       # 基金名称
├── amount: number          # 每期金额（元）
├── frequency: "weekly" | "biweekly" | "monthly"  # 频率
├── day: number             # 周几(1-7) 或 几号(1-28)
├── status: "active" | "paused" | "stopped"  # 状态
├── created_at: string      # 创建时间
├── last_executed: string   # 上次执行时间
└── total_executed: number  # 已执行次数
```

### 2.3 扩展：FundHolding 新增字段

```
├── total_shares: number       # 累计持有份额
├── total_cost: number         # 累计投入成本
├── avg_nav: number            # 平均成本净值 = total_cost / total_shares
├── transaction_count: number  # 交易笔数
```

### 2.4 localStorage 数据结构总结

```
fund_assistant_portfolio       # 持仓列表（现有，扩展字段）
fund_assistant_transactions    # 交易流水 Transaction[]
fund_assistant_auto_invest     # 定投计划 AutoInvestPlan[]
fund_assistant_config          # 配置（现有）
```

## 三、加减仓功能设计

### 3.1 交互方式一：卡片按钮操作

在每张持仓卡片上新增 **加仓/减仓** 按钮组：

```
┌─────────────────────────────────────┐
│  易方达蓝筹精选  005827      观望 ⏸️ │
│  成本 ¥20,000 → 市值 ¥21,400  +7%  │
│  盘中估值 +0.35%                     │
│                                      │
│  [➕ 加仓]  [➖ 减仓]       [删除]   │
└─────────────────────────────────────┘
```

- 按钮默认隐藏，**点击卡片选中后**才显示（和现有删除按钮一致）
- 点击后弹出**底部弹窗（半屏 drawer）**

#### 加仓弹窗

```
┌─────────────────────────────────┐
│  ➕ 加仓 · 易方达蓝筹精选        │
│                                  │
│  金额（元）                      │
│  ┌──────────────────────────┐   │
│  │ 5000                     │   │
│  └──────────────────────────┘   │
│                                  │
│  快捷金额：                      │
│  [500] [1000] [2000] [5000]     │
│                                  │
│  成交净值：1.8520（最新净值）     │
│  预估份额：2699.78 份            │
│                                  │
│  备注（可选）                    │
│  ┌──────────────────────────┐   │
│  │                          │   │
│  └──────────────────────────┘   │
│                                  │
│       [ 确认加仓 ]              │
└─────────────────────────────────┘
```

#### 减仓弹窗

与加仓弹窗类似，额外展示当前持有份额和市值，防止卖超：
- 输入金额后实时计算赎回份额
- 金额不能超过当前市值，超出时按钮置灰 + 提示

### 3.2 交互方式二：自然语言操作

底部输入栏支持自然语言加减仓，示例：

- "易方达蓝筹加仓5000"
- "白酒减仓3000元"
- "005827买入2000"

后端 `parse_natural_language` 扩展识别意图：
- 现有能力：识别基金名/代码 + 持仓信息 → **新增持仓**
- 新增能力：识别"加仓/买入/减仓/卖出" + 金额 → **交易操作**

解析结果新增 `intent` 字段：

```
{ intent: "add_holding" }              # 新增持仓（现有）
{ intent: "buy", amount: 5000 }        # 加仓
{ intent: "sell", amount: 3000 }       # 减仓
```

### 3.3 交易后的数据更新流程

```
用户操作（按钮/自然语言）
  → 生成 Transaction 记录，存入 localStorage
  → 重新计算该基金的累积持仓：
      total_shares += (buy) 或 -= (sell)
      total_cost += (buy) 或 -= (sell 按比例)
      avg_nav = total_cost / total_shares
      cost = total_cost（更新卡片显示）
  → 刷新持仓卡片
  → Toast 提示"加仓成功 +5000元"
```

### 4.4 自动执行逻辑

定投的"自动模拟执行"在前端侧判断 + 后端辅助：

```
用户打开页面（或页面定时检查）
  → 遍历所有 status=active 的定投计划
  → 计算从 last_executed 到 now 之间有几期未执行
  → 如果 未执行期数 = 1：
      静默自动执行
  → 如果 未执行期数 > 1：
      弹出确认提示条（页面顶部横幅）：
      ┌──────────────────────────────────┐
      │ 📅 有 3 期定投待执行（共¥1500）   │
      │ [查看详情]  [全部执行]  [跳过]    │
      └──────────────────────────────────┘
      
      点"查看详情"→ 展开每期明细
      点"全部执行"→ 按各期对应日期的净值逐笔生成交易记录
      点"跳过"→ 跳过这些期数，last_executed 更新到当前
```

#### 下次执行时间计算规则

- **每周**：从 `last_executed` 往后推到下一个目标周几
- **每两周**：从 `last_executed` 往后推 14 天到目标周几
- **每月**：从 `last_executed` 往后推到下个月的目标日

#### 边界处理

- **非交易日（周末/节假日）**：顺延到下一个交易日执行
- **用户多天没打开页面**：打开时计算所有未执行期数，弹出确认横幅，用户确认后按各期历史净值逐笔批量执行
- **暂停期间**：不执行，恢复后从下一个周期开始

### 4.5 定投状态管理

| 操作 | 状态变化 | 效果 |
|------|---------|------|
| 创建 | → active | 开始按周期执行 |
| 暂停 | active → paused | 暂停执行，保留计划 |
| 恢复 | paused → active | 从下一个周期继续 |
| 停止 | → stopped | 永久停止，计划归档不删除 |

## 六、API 变更

后端只需新增一个辅助接口，用于获取历史净值（定投补执行时需要）：

```
GET /api/fund/{fund_code}/nav-history?start=2026-03-01&end=2026-04-02

Response:
{
  "fund_code": "005827",
  "nav_list": [
    { "date": "2026-03-26", "nav": 1.8310 },
    { "date": "2026-04-02", "nav": 1.8520 }
  ]
}
```

其余逻辑（交易记录、定投计划、累积计算）全部在前端 localStorage + JS 完成。

### 自然语言解析扩展

`parse_natural_language` 返回结果新增 `intent` 字段：

```json
{ "intent": "add_holding", ... }           // 新增持仓（现有）
{ "intent": "buy", "amount": 5000, ... }   // 加仓
{ "intent": "sell", "amount": 3000, ... }  // 减仓
```
